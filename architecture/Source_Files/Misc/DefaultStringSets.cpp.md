# Source_Files/Misc/DefaultStringSets.cpp

## File Purpose

Provides compiled-in default string sets for the Aleph One game engine, serving as fallback localized text when no MML file provides custom strings. Uses static object instantiation to register strings with the TextString system at link/load time, eliminating the need for external resource files in minimal distributions.

## Core Responsibilities

- Register default error messages, UI labels, menu items, and game-related text
- Provide strings for network dialogs, difficulty levels, game modes, and team colors
- Supply key name mappings for input handling (e.g., "Return", "Escape")
- Support both legacy MacOS STR# resource strings and modern SDL widget string arrays
- Enable fallback gameplay when no external string resources are available

## Key Functions / Methods

### AutoStringSetBuilder (constructor)

- **Signature**: `AutoStringSetBuilder(short inStringSetID, const char** inStrings, short inNumStrings)`
- **Purpose**: Register a string array with the TextString repository by iterating through each string and calling `TS_PutCString`.
- **Inputs**: 
  - `inStringSetID`: Numeric ID identifying the string set (e.g., 128, 146)
  - `inStrings`: Pointer to array of C-style null-terminated strings
  - `inNumStrings`: Count of strings in the array
- **Outputs/Return**: None
- **Side effects**: Invokes `TS_PutCString(inStringSetID, i, inStrings[i])` for i Γêê [0, inNumStrings). Populates the global TextString repository. Executes at static initialization time.
- **Calls**: `TS_PutCString` (defined in TextStrings.h)
- **Notes**: This class has no instance state; it exists purely for registration side effects. The constructor is called exactly once per BUILD_STRINGSET macro invocation.

## Macros

**NUMBER_OF_STRINGS**: Computes array length via `sizeof(sa) / sizeof((sa)[0])`.

**BUILD_STRINGSET**: Expands to `static AutoStringSetBuilder (strs ## Builder) (id, strs, NUMBER_OF_STRINGS(strs))`, declaring a static builder instance that registers the string array at link time.

## Control Flow Notes

This file contains no dynamic control flowΓÇöit is a data-and-initialization file. All string registration occurs automatically during C++ static object initialization before `main()` is called. String sets are logically organized by function (errors, network, items, UI, colors), reflecting original MacOS STR# resource groupings from Bungie's Marathon.
